Output: A working context management system with file-based persistence and git versioning, supporting named algorithms and session recovery.

  <action>
Create `.algomath/state.py` with the flexible branching state machine per D-06, D-07, D-08.

State definitions per D-07:
```python
class WorkflowState(Enum):
    IDLE = "idle"
    TEXT_EXTRACTED = "text_extracted"
    STEPS_STRUCTURED = "steps_structured"
    CODE_GENERATED = "code_generated"
    EXECUTION_COMPLETE = "execution_complete"
    VERIFIED = "verified"
```

Create `SessionState` class tracking:
- `current_algorithm: str` — name of active algorithm (or None for session-only)
- `current_state: WorkflowState` — current workflow state
- `data: dict` — current algorithm data (text, steps, code, results)
- `can_transition_to(target: WorkflowState) -> bool` — check if transition is valid

Transitions (flexible branching per D-06):
- User can jump between states if data exists
- From any state, can go back to previous states
- Forward transitions require data to exist

Create `.algomath/context.py` with `ContextManager` class:

Methods:
1. `start_session()` — initialize new session, load or create session state
2. `get_current() -> SessionState` — return current session state
3. `save_text(text: str, name: str = None)` — save extracted text, move to TEXT_EXTRACTED
4. `save_steps(steps: list)` — save structured steps, move to STEPS_STRUCTURED
5. `save_code(code: str)` — save generated code, move to CODE_GENERATED
6. `save_results(results: dict)` — save execution results, move to EXECUTION_COMPLETE
7. `mark_verified()` — mark as verified
8. `load_algorithm(name: str)` — switch to named algorithm
9. `create_algorithm(name: str)` — create new named algorithm
10. `get_progress() -> dict` — return progress info for display per D-09, D-10

Session persistence:
- Save session state to `.algomath/session.json` on every change
- Auto-load on start_session() if exists
- Per D-18/D-21: Support both named (persisted) and session-only (unnamed) algorithms
